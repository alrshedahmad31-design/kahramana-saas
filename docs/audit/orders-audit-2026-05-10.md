# Orders Audit - 2026-05-10

Scope:
- `src/app/[locale]/dashboard/orders/**`
- `src/components/orders/**`
- server actions, RLS policies, RPCs, and dashboard components used by the orders section

Notes:
- Read-only audit. No application code was changed.
- No direct Riffa/Qallali cross-branch leak was found in the main orders list or `updateOrderStatus`; both have branch scoping through app logic and/or RLS. The main security failures are role-scope bypasses and mutation integrity gaps.
- Realtime subscriptions found in this scope call `removeChannel`; the one risky pattern is subscription churn in `OrderDriverMap`, not a missing cleanup.

## Top 5 Critical Fixes By Impact

1. Add `requireDashboardSection('orders')` to the order detail route and modal detail action, then assert branch scope after fetching.
2. Make `updateOrderWithReason` use the same server-side transition and branch/role validator as `updateOrderStatus`.
3. Verify update row counts for both status actions so concurrent changes cannot return false success.
4. Tie cancellations/returns to payment/refund state before changing terminal order status.
5. Handle Supabase errors in orders list/detail queries and surface failures instead of rendering empty data or 404.
