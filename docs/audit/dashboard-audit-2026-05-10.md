# Dashboard Audit - 2026-05-10

Scope: `/dashboard/**`, `/waiter`, `/driver`, `/pos`.

Notes:
- Read-only audit. No application code was changed.
- No client-side exposure of `SUPABASE_SERVICE_ROLE_KEY` was found in the scanned dashboard/waiter/driver/POS client components. Service-role usage found in this scope is server-side.
- Realtime listeners found in this scope generally call `removeChannel` on unmount; no uncleaned realtime subscription was identified.
- Top-level `/pos` is not implemented; the existing POS route is `/dashboard/pos`.

## Top 5 Critical Fixes By Impact

1. Lock down `/dashboard/shifts` server actions and RLS branch scope. This is the highest-impact cross-branch data-integrity and RBAC risk.
2. Fix waiter RBAC consistency so the `waiter` role can either complete its intended workflow or is removed from route access.
3. Add a server-side section guard to `/dashboard/audit` and align audit-log RLS with owner/GM-only audit visibility.
4. Make POS/waiter order creation transactional for required payment and audit records, or explicitly fail when those records cannot be written.
5. Add server-side validation for shift closing cash values and branch IDs before financial records are persisted.
